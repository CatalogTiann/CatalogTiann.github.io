Python虚拟环境的作用及搭建方法

发表于 2017-10-28 | 分类于 Python |
字数统计: 956 | 阅读时长 ≈ 3

为什么需要虚拟环境

在我们日常的项目开发过程中, 我们通常会按照项目的需求去下载相应的框架库, 如 Django, request 等, 但是不同的项目所使用的框架库并不一定是相同的, 或者使用相同的框架但是版本需求是不同的, 这时候需要我们根据需求不断的安装, 更新或卸载相应的库. 会对我们的开发环境和项目造成很多不必要的麻烦, 管理起来也相当困难.

想象一下下面这种场景:

项目A 需要 Django 1.11.9 版本, 项目B 需要 Django 1.6.5 版本. 如果没有不使用虚拟环境, 那么当你运行这两个项目调试时, 你就需要重复安装 Django 1.11.9 版本调试项目A, 然后卸载, 安装 Django 1.6.5 版本调试项目B, 我敢打赌, 折腾两次你就炸毛了

什么是虚拟环境

多个虚拟环境共用一个 Python 解释器, 给我的感觉就像是多个操作系统共用一套硬件设施.

每一个虚拟环境都相当于一个新的 Python. 你可以在这个新的环境里安装三方库, 环境与环境之间是相互隔离的, 也就是说在 A 环境中安装的库只在 A 中使用, 不会影响其他环境.

总结：

  1. 虚拟环境可以让系统 Python 环境保持纯洁, 干净, 避免包的混乱和版本的冲突.
  2. 专为不同的项目搭建不同的环境, 为每个项目使用虚拟环境可以保证项目只访问虚拟环境中的包, 从而保持全局解释器的干净整洁, 使其只作为创建虚拟环境的源.
  3. 使用虚拟环境不需要管理员权限.
  4. 多个虚拟环境同时进行, 如同你拥有了多个 Python 解释器.

虚拟环境的搭建与使用

准备工作

打开 Terminal/cmd, 输入 pip3 install virtualenv
(本人已经配置了虚拟环境，全局环境中已经有了virtualenv包，所以在虚拟环境中给大家演示安装过程)

创建一个虚拟环境

打开 Terminal/cmd, 先切换到一个你喜欢的目录下, 因为 virtualenv 会在当前目录下创建一个虚拟环境, 这里我选择直接创建了一个 My_venv 的文件夹, 然后在该文件夹下进行虚拟环境搭建.

输入 virtualenv + 空格 + 环境名称, 就可以生成一个干净的 Python 环境, 这里我们取名叫 venv1

使用虚拟环境

激活虚拟环境

ps: 只有激活之后，才算进入该虚拟环境，否则安装包时，依然是安装在全局环境之下

对于Mac: 进入虚拟环境所在目录, 该目录下有一个 bin 文件夹, 进入bin, 输入 source activate 激活虚拟环境, 当游标前出现括号包裹的虚拟环境名称时, 激活成功. 

退出方法: 输入 deactivate 或者退出当前 terminal

对于Windows: 激活文件在 Script 文件夹下. 激活方法: 进入 Script 文件夹下输入 activate

退出方法: 输入 deactivate 或者退出当前 cmd

进入虚拟环境以后, 就可以安装项目所需库, 搭建专属于项目的环境.

在 Pycharm 中使用虚拟环境

然后选择你创建的虚拟环境所在目录下的 python 解释器就可以使用该虚拟环境了.

当项目不再需要维护时, 就可以直接删除虚拟环境所在文件夹, 就相当于删除了当前虚拟环境. 简单方便的同时还保证了系统 Python 环境的干净.
